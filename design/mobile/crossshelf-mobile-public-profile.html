Public profile design — reduces overlap with Library

  • This IS a Library shape with an identity header layered on top. Looking at your desktop, you were right that the two pages duplicate each other heavily. The architectural model: "viewing your own collection (Library) vs viewing someone else's collection (Public Profile)." Same shape, different ownership and affordances.
  • Identity block — large avatar, name, joined date, member rank badge (gold star + "#2" for Claudoro27). For founding members (member #1 through some early threshold) a "FOUNDING MEMBER" pill would appear below the rank. For later members, just the rank number.
  • Bio field below the identity block when filled in. Quote-styled with subtle teal left border, italic text. Shown only when the user has written a bio — empty bios get hidden entirely. Edit Profile lets the user write it as a multi-line text area.
  • Follow button top-right of the identity block. Primary action when viewing someone else. When viewing your own profile, this is replaced with "EDIT PROFILE" in muted styling.
  • Taste fingerprint section right under the identity block. Same type-mix bar from the People page (40% Movies / 40% TV / 20% Books for Claudoro27) but at full width with a small color legend below since this is the showcase placement of the feature. Visual answer to "what does this person review?"
  • Stats row — 5 stats laid out evenly (Rated / Reviewed / Tracked / Followers / Following) with color-coded numbers matching the platform color language. Compact, scannable, gives you the shape of someone's CrossShelf presence at a glance.
  • Status pills identical to Library — All / Done / Going / Want / Drop. Tap to filter sections below. Dropped section IS shown on public profile per user decision — letting users own their drops is part of being a reviewer ("I'm not afraid to drop bad shows").
  • Filter row simpler than Library — type pills + sort button, but NO search pill. Reasoning: searching within another user's library is unusual behavior; their library is smaller and scrolling is fine. Could add later if users request it.
  • "Top Rated" section — items the user gave 5 stars to, regardless of status. This is the "best of" section your desktop has. Shows their highest opinions. Important for "who is this reviewer" framing.
  • Per-status sections below exactly like Library — Completed / In Progress / Want To. Same card design, same color language, same progressive expansion. The only difference: no edit affordances (you can't change Claudoro27's status of an item).
  • Reviews section at the bottom — their actual review text with item context. Each card shows a small cover thumbnail + item title (tappable to item detail) + media type + date + their star rating + review text + upvote/downvote/reply with thread counts. This is where the "reviewer identity" really lives — what they think, not just what they watched.
  • Top header with back button + @username for arriving from People page. Share icon on the right lets users share a profile link.

What's different when viewing YOUR OWN profile

  • FOLLOW button replaced with EDIT PROFILE button in muted (non-teal) styling — secondary action.
  • "FOUNDING MEMBER" pill appears for early-rank members (Member #1 through some threshold). For Juan as #1, this would show prominently.
  • Could surface management affordances like "Add review" or "Share to friend" but the natural path is: edits happen in Library, viewing happens here. Profile stays read-mostly even when it's yours.
  • Search pill could optionally appear on your own profile since searching your own library is a useful behavior.

What's different from Library

  • Identity block + bio + stats + taste fingerprint added at top.
  • "Top Rated" section added above status sections.
  • Reviews section added at bottom.
  • All status sections shown including Dropped (subject to user's Private Library toggle).
  • No search pill, no import button, no Following franchises section. These are management affordances that only matter when it's your own data. (Following franchises could optionally appear with a "Tracks" label — but typically users want their franchises private.)
  • No edit affordances on cards. Tapping a card still goes to item detail, but you can't change the status of items in someone else's library.

Privacy model (locked — Option B)

  • Single "Private Library" toggle on Edit Profile. Matches the existing desktop pattern.
  • When enabled, hides everything except identity block + bio. Top Rated section hides, all status sections (Completed/In Progress/Want/Dropped) hide, Reviews hide, taste fingerprint hides, stats row hides. Only identity block + bio remain.
  • Clear "Library is private" indicator card replaces the hidden content. Lock icon + brief explanation + suggestion to follow them for activity feed. Honest with visitors about why the page looks sparse.
  • Reviews hide too (decided as Option B). Even though reviews are intentional public publication, users who hide their library probably want consistency — they don't want strangers reading their opinions if they didn't even want strangers seeing what they collected. Single toggle is simpler than separate toggles for different content types.
  • Follow button stays active on private profiles. Users can still follow each other based on identity/bio alone, and following gives them access to that user's activity in the People feed (a softer disclosure).
  • CRITICAL pre-launch verification: The privacy toggle must enforce privacy at the API/database level, not just hide UI. If someone queries /u/[username] or any library/review/stats endpoint, the backend must check the toggle and return nothing if private. UI-only privacy would be a real violation.
  • Per-section privacy could be a future feature if users ask for it (hide reviews but show library, or vice versa). Not implementing now — single toggle is simpler and matches user mental model of "all in or all out."
