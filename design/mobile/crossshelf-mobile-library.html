What's different in v2 (matches desktop architecture)

  • Per-status sections stacked vertically — Completed / In Progress / Want To / Dropped each get their own section with their own header and count, matching desktop layout. Default view shows all sections. Tapping a status pill at the top filters to just that section.
  • Status pills colored per status (teal/blue/purple/red) matching desktop's colored buttons. Each shows a count. "All" pill on the far left for the default view.
  • Following section at the top showing franchise/series the user is tracking. Horizontal scrolling cards with multi-thumb collages (matches your desktop layout), franchise title, type breakdown badges, "Rated X / Y" progress with thin red bar. Highlights cross-media tracking — a real differentiator.
  • Search pill styled distinctly from filter pills. Matches the sort button (4px square corners, teal-tinted background and border) instead of the round filter pills (14px). Visually groups search + sort as "tools" — the round pills in the middle are filters. Tap to expand into a search input that searches within library only.
  • Each section shows 4 cards by default. The "Show 4 more" button at the bottom of each section expands the visible set progressively — tap once to see 4 more (total 8), tap again for 4 more (total 12), keep tapping until you've seen everything. The remaining count is shown next to the action ("Show 4 more · 11 remaining") so the user knows how much is left.
  • Show-more button is a real button, not text. Styled like a tappable pill with chevron icon, teal-tinted. Reads as an action affordance. Different from the discreet "+ N more →" text I had before — the prior version felt more like a status indicator than something interactive.
  • Sort button visually distinctive — different shape (4px corner radius vs 14px pills), teal-tinted background/border. Same treatment now applied to search pill so search and sort visually group as "tools."
  • Cards match desktop style — bright colored type badge in top-left, gold rating badge in top-right with star + number, title below, score number + percentage bar + percentage, year + genres, your gold star rating at the bottom. Poster-tall aspect ratio (2/3) gives covers real visual weight.
  • Type badge color language locked: red Movies, yellow Games, teal Manga, blue Books, pink Anime, green TV, purple Podcasts, orange Comics — matches your desktop badges exactly.
  • Two-column grid for item cards on mobile. Three would be too cramped at 380px, one would feel like a list when these are visual posters.
  • "+ N more [status] →" affordance at the bottom of each section keeps the initial page scrollable, lets users dig in to a specific section if they want more.
  • Status pill labels shortened — "Done / Going / Want / Drop" instead of the full words to fit on one row without scroll. Section headers below still use full labels.
  • In Progress + Want To + Dropped cards show different info — In Progress shows progress ("Ep 8 / 15", "p. 145 / 545"). Want To shows when added. Dropped shown at 0.7 opacity for visual hierarchy.

Interaction patterns

  • Tap status pill → filters to show only that status section. Tap "All" or active pill to deselect.
  • Tap type pill → filters within current status. Multiple types can be active.
  • Tap search pill → expands to input for searching within library only.
  • Tap sort button → cycles through Recent / Highest rated / A-Z / Recently added.
  • Tap card → opens item detail page.
  • Tap "+ N more" → expands that section inline.
  • Tap Following card → opens the franchise page (/franchise/[slug]).
  • Tap Import → opens import flow.
