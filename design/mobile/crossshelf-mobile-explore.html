Explore page design

  • Search bar is the primary action — large, prominently placed below the page title. Placeholder communicates scope ("titles, genres, vibes, people…") so users know it searches multiple dimensions. Tapping opens a full-screen search experience (not designed yet — would be a separate small mockup).
  • Type filter chips with counts — single non-scrolling row, all 9 types visible at once. Each chip color-coded to match the type's identity. Compact vertical layout (label on top, count below) lets all 9 fit at 380px without horizontal scroll. Numbers tell users what's actually in the catalog. Tap a chip to filter; tap again to deselect.
  • "Building" catalog indicator on Music and Podcasts chips — small muted gold dot in the top-right corner of those chips. Honest signal that those catalogs are smaller / actively growing. The catalog IS real (200 albums, 475 podcasts) so the chips aren't dimmed or hidden, just marked. On their per-type section headers, a small "BUILDING" pill in muted gold spells out the framing more explicitly.
  • Filters button + applied state + sort on a single row. The "FILTERS" button is teal-tinted (matches the sort/search styling from Library). When filters are applied, the inline text updates: "Showing all" → "Filters: Dark, Sci-Fi" so users know what's active without opening the sheet. Sort label on the right shows current sort.
  • Filter sheet (not shown but spec'd) — Tapping FILTERS opens a bottom sheet with three filter dimensions from desktop: Genre, Vibe, Tag. Each with multi-select chips. Apply button closes the sheet and updates the page.
  • Per-type sections follow the For You pattern — section header with type name + count + "See all →", then horizontal-scrolling card row. Familiar layout from For You.
  • Cards are smaller than For You's cross-shelf cards (110px wide) so 3 full cards fit per scroll position. Visual density matches the Library card style but tighter.
  • Multi-type badges shown when applicable. Hunter x Hunter and Fullmetal Alchemist show both "TV" + "ANIME" badges, matching desktop's pattern. Items can belong to multiple types.
  • "See all →" on each row goes to a dedicated type-browse page (e.g. /explore/movies) with grid layout and full filter set.

Where this differs from desktop deliberately

  • Three filter rows compressed into one filter button. Desktop's "Genre / Vibe / Tag" three-row layout would consume most of mobile's vertical real estate. Bottom sheet preserves the same filter dimensions while leaving the main page for content.
  • Type pills compressed into a non-scrolling row of 9 — all visible at once instead of horizontal-scroll, by going text-only with stacked counts. Loses the icons from desktop, but keeping all 9 types visible without scroll is the bigger win for discoverability.
  • Type sections start immediately instead of after a deeper filter UI. Default state shows content. Users can drill in if they want.
  • Search prominence increased. On desktop it's one of many controls. On mobile, search is the obvious primary action since browsing is harder than searching.

When a type is selected

  • Tapping "Movies" chip filters to only show the Movies section, expanded — instead of horizontal scroll, the section becomes a 2-column vertical grid of all movies. Other type sections hidden. The filters button still works for narrowing by genre/vibe/tag within Movies.
  • Tapping the active Movies chip again returns to default (all types shown in horizontal rows).

Interactions to design later

  • Search results page — what tapping the search bar opens. Could be its own mini-mockup.
  • Filter sheet UI — the bottom sheet with Genre/Vibe/Tag selection. Could be its own mini-mockup.
  • Per-type browse page — what "See all →" leads to. Probably a grid layout matching Library's card style.
